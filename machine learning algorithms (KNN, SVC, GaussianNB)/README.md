У ході виконання роботи було розглянуто роботу кількох моделей класифікації на наборі даних рукописних цифр. Дані складаються з 1797 зображень розміром 8×8 пікселів, які були поділені на навчальну та тестову вибірки у пропорції 80% до 20%, тобто 1437 прикладів для навчання та 360 для тестування.
Після навчання моделі KNearest Neighbors (KNN) з параметрами за замовчуванням було проведено прогнозування на тестових даних. Перші 20 значень прогнозу показали майже повне співпадіння з очікуваними результатами: помилка зустрілася лише один раз. Загальна точність моделі склала 98.61%, що свідчить про дуже ефективне розпізнавання цифр. Матриця невідповідностей показала, що більшість цифр класифікуються абсолютно правильно. Лише окремі випадки спостерігалися для цифр, які дійсно мають схожі форми, наприклад, 3 іноді плуталась із 5 або 7, а 4 - інколи з 7. Усі інші класи були розпізнані майже безпомилково.
Детальні показники classification report підтвердили високий рівень роботи моделі: для більшості цифр показники precision, recall і f1-score наближені або рівні 1.00. Загальна точність становить 0.99, що свідчить про дуже надійну і стабільну роботу алгоритму KNN на цьому датасеті.
Було проведене порівняння з іншими моделями. Модель Support Vector Classifier показала таку саму точність, як і KNN - 98.61%. Це очікуваний результат, адже SVC часто показує високу якість на таких структурах даних. Натомість Gaussian Naive Bayes показав значно нижчу точність - лише 85.56%, що пояснюється тим, що цей алгоритм припускає нормальний розподіл ознак, тоді як значення пікселів зображень не відповідають цьому припущенню. Відповідно, Байєсівський класифікатор не є оптимальним для цього завдання.
Було проведено підбір найкращого значення параметра K для алгоритму KNN. Результати тестування для значень K від 1 до 20 показали, що найвищу точність забезпечує K = 2, при якому точність досягає 99.44%. Це найкращий результат серед усіх досліджених моделей. Для більших значень K точність поступово зменшується, оскільки модель стає занадто узагальненою і менш чутливою до локальних особливостей даних.
